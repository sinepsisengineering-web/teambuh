# Системные требования TeamBuh

## Требования к серверу

### Аппаратные требования

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| **CPU** | 2 ядра | 4 ядра |
| **RAM** | 2 ГБ | 4 ГБ |
| **Диск** | 10 ГБ SSD | 50 ГБ SSD |
| **Сеть** | 10 Мбит/с | 100 Мбит/с |

### Программные требования

| Компонент | Версия | Обязательно |
|-----------|--------|-------------|
| **Node.js** | 18.0.0+ | ✅ Да |
| **npm** | 9.0.0+ | ✅ Да |
| **Операционная система** | Windows Server 2019+ / Ubuntu 20.04+ / macOS 12+ | ✅ Да |
| **Python** | 3.x (для компиляции native модулей) | ⚠️ Для Windows |
| **Visual Studio Build Tools** | 2019+ (для better-sqlite3) | ⚠️ Для Windows |

---

## Зависимости приложения

### Серверные (backend)

Устанавливаются на сервере, выполняются в Node.js:

```
express           - Веб-сервер API
cors              - Кросс-доменные запросы
multer            - Загрузка файлов
better-sqlite3    - SQLite база данных (требует компиляции)
```

### Клиентские (frontend)

Компилируются в статические файлы, выполняются в браузере:

```
react             - UI библиотека
react-dom         - Рендеринг React
vite              - Сборщик (только разработка)
tailwindcss       - CSS фреймворк
```

---

## Сетевые требования

### Порты

| Порт | Назначение | Когда нужен |
|------|------------|-------------|
| **3001** | API сервер | Всегда |
| **5173** | Dev сервер Vite | Только разработка |
| **80/443** | Веб (через nginx) | Продакшен |

### Домены/SSL

- Для продакшена рекомендуется HTTPS (Let's Encrypt)
- API и frontend могут быть на разных доменах (CORS настроен)

## Хранение данных

### Объём данных (оценка)

| Данные | На 40 клиентов/год | На 5 лет |
|--------|-------------------|----------|
| Задачи (SQLite) | ~1.5 МБ | ~8 МБ |
| Клиенты (JSON) | ~100 КБ | ~200 КБ |
| Сотрудники (JSON) | ~50 КБ | ~100 КБ |
| Документы (файлы) | ~500 МБ | ~2.5 ГБ |
| **Итого** | **~500 МБ** | **~3 ГБ** |

### Бэкап

Рекомендуется ежедневный бэкап папки:
```
data/tenants/org_default/
```

## Масштабирование

### Текущая архитектура

- Однопользовательский/малая команда (до 10 пользователей)
- Один сервер
- Файловое хранение + SQLite

### При росте (>100 клиентов, >10 пользователей)

- [ ] Переход на PostgreSQL
- [ ] Отдельный файловый сервер
- [ ] Балансировка нагрузки
- [ ] Redis для кэширования
